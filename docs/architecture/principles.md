# Princípios Arquiteturais

1. Núcleo pequeno e estável.
2. Bordas dependem do centro; centro não depende das bordas.
3. Adapters concretos nunca invadem regras centrais.
4. Avatar é apresentação, não cérebro.
5. LLM sugere; policy decide.
6. Memória é local-first e auditável.
7. Tela e desktop são recursos sensíveis.
8. Skills são capacidades governadas, não atalhos.
9. Configuração é separada de segredo.
10. Testes protegem comportamento e fronteiras.
11. Documentação registra decisões reais.
12. O projeto deve ser neutro e personalizável para uso público.
