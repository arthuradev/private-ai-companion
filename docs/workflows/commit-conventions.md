# Convenções de Commit

Use Conventional Commits.

## Tipos

- `feat`: nova funcionalidade.
- `fix`: correção.
- `docs`: documentação.
- `test`: testes.
- `refactor`: refatoração sem mudança funcional.
- `chore`: tarefas de manutenção.
- `build`: empacotamento/build.
- `ci`: integração contínua.

## Exemplos

```text
feat(core): add event bus
feat(memory): add sqlite repository
fix(safety): block critical shell actions
docs(adr): record avatar adapter decision
test(brain): add prompt builder snapshots
```
