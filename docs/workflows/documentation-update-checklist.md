# Checklist de Atualização de Documentação

Atualize documentação quando mudar:

- arquitetura;
- boundaries;
- módulos;
- eventos;
- segurança;
- memória;
- skills;
- configuração;
- fluxo de usuário;
- comando de instalação/execução;
- processo de release.

Documentos possíveis:

- `ARCHITECTURE.md`
- `SDD.md`
- `SECURITY.md`
- `docs/modules/*.md`
- `docs/architecture/adr/*.md`
- `docs/configuration/*.md`
- `CHANGELOG.md`
